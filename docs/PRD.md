# 產品需求文件 (PRD) - AI 學習助理系統

## 1. 專案概述
- **背景與動機**：現代學生往往面臨繁重的課業壓力與資訊過載的問題，難以有效率地消化知識並檢驗自身學習成效。為了提升學習效率，我們希望開發一套「AI學習助理系統」，利用AI技術協助學生自動整理筆記、生成客製化測驗題目，並追蹤學習成果，打造個人化的學習體驗。
- **目標用戶**：學生（包含國高中生、大學生及準備各類考試的學習者）。
- **核心價值主張**：透過AI自動化處理繁瑣的筆記整理與出題工作，讓學生能專注於內容理解與弱點補強，大幅節省備課與複習時間，提升整體學習成果。

## 2. 功能需求

### 2.1 自動整理筆記
- **使用者故事**：作為 **學生**，我希望 **能夠上傳或輸入零散的課堂筆記文字**，以便 **AI能自動幫我歸納、排版並產生結構化的重點摘要**。

### 2.2 AI 出題與測驗
- **使用者故事**：作為 **學生**，我希望 **系統能根據我的筆記內容或指定單元，自動生成單選題或問答題等測驗**，以便 **我能隨即進行自我測驗，並即時獲得解答與詳細解析**。

### 2.3 學習進度追蹤
- **使用者故事**：作為 **學生**，我希望 **系統能記錄我每次學習的時間、單元以及測驗成績**，以便 **我能透過視覺化圖表或清單，隨時了解自己的學習軌跡與達成率**。

### 2.4 弱點分析建議
- **使用者故事**：作為 **學生**，我希望 **系統能統整分析我在測驗中容易答錯的題型與概念**，以便 **AI能針對我的薄弱環節提供重點複習計畫與後續學習建議**。

### 2.5 語音問答學習
- **使用者故事**：作為 **學生**，我希望 **能夠透過語音的方式與AI助理進行對話互動**，以便 **在不方便打字或需要情境對話（如語言練習、快速問答）時，也能順暢地提問與學習**。

## 3. 非功能需求
- **技術限制**：
  - 後端採用 Flask 框架。
  - 前端使用 Jinja2 模板渲染（搭配基本 HTML/CSS/JS）。
  - 資料庫使用 SQLite，確保輕量與便於本地開發。
- **效能與安全考量**：
  - AI 整理與出題需具備良好的錯誤處理機制，並提示使用者等待時間。
  - 學生的筆記與問答紀錄屬個人資料，需實作基礎的存取控制，未經授權無法檢視他人資料。

## 4. MVP 範圍

### Must Have (核心必要功能)
- 自動整理筆記（純文字上傳與產出重點摘要）。
- AI 出題與測驗（根據筆記產生單選題，並能選擇正確答案看解析）。
- 使用者註冊/登入與基礎資料庫建置。

### Should Have (重要功能)
- 學習進度追蹤（文字清單顯示歷史成績與測驗紀錄）。
- 弱點分析建議（AI 從錯題清單中生成一段短文字建議）。

### Nice to Have (加分期望功能)
- 語音問答學習（整合 Web Speech API 進行語音輸入與語音合成回應）。
- 圖表化（Dashboard）的個人學習能力分佈。
- 可匯出測驗結果與筆記為 PDF 檔案。
